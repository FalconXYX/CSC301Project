-Correctly formulated user stories 
All stories are in github project board and follow user story template. Refer to CSC301-PhysioFind-Project-Tracking.png.

-Broken down into tasks
Sprint is broken down into tasks and assigned to team members. Refer to CSC301-PhysioFind-Project-Tracking.png. and Defined_Acceptance_Criteria_Example.png.
Each sprint is labelled as X.Y.Z where X is the sprint number, Y is the epic number and Z is the task number.
For example, 2.1.1 is the first task of the first epic of the second sprint.

-Estimated
There is a number attached to each ticket in the github project board. That is the estimated story points for that ticket.
Refer to CSC301-PhysioFind-Project-Tracking.png.

-Prioritized
Each ticket is prioritized as P0, P1, P2, P3, P4 or P5. Refer to CSC301-PhysioFind-Project-Tracking.png.

-Clearly Defined Acceptance Criteria
All tickets have clearly defined acceptance criteria. Refer to Defined_Acceptance_Criteria_Example.png.
